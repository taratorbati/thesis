2. **Infrastructure** (`src/`): terrain graph builder with DEM-padded fractional routing boundary conditions (terrain.py v2.0), parquet I/O with skip-if-exists, precomputation cache, perfect/noisy forecast providers, generic `run_season()` runner.

3. **Fractional routing fix (terrain.py v2.0):** DEM padded by 1 cell on each edge, slope extrapolated. `Nr` counts ALL lower neighbors (internal + external); `sends_to` lists only internal ones. ABM divides phi2 by Nr → off-farm water exits the system automatically. Fixed the "bathtub effect" where 3 sink agents trapped all cascade runoff.

4. **Four controllers implemented:**
   - No-irrigation baseline (3 runs complete)
   - Fixed-schedule with 19-event linear-decay (9 runs complete: 3 scenarios × 3 budgets)
   - **MPC via CasADi+IPOPT with smooth approximations (27 runs complete: 3 scenarios × 3 budgets × 3 horizons {Hp=3, 8, 14}, all with perfect forecast)**
   - SAC RL — Gymnasium wrapper, SB3 training script, Kaggle notebook (built, NOT yet trained)

5. **Ponding penalty (cost.py v2.2):** Field-mean across all 130 agents (not sink-only). X5_REF=50mm.

## Three Pathologies Identified in the 27 MPC Runs (Diagnostic Chain)

**FC-Overshoot pathology (dry/100%/Hp=14):** 21.6 waterlog-days/agent. The optimizer parks x1 above FC=140mm despite no rainfall reason to do so. Root cause: indirect h6 stress (multiplicative on biomass increment) provides too weak a disincentive against over-irrigation when water is cheap.

**Hp Inversion pathology (wet scenario):** Hp=3 outperforms Hp=8 outperforms Hp=14 on biomass (880 vs 857 vs 808 g/m²) with perfect forecast. Counterintuitive — longer horizon should never do worse with perfect forecast.

**Chronic waterlogging pathology (wet/100%/Hp=14):** 61 waterlog-days/agent, x1 stays at 175-205mm for 25+ consecutive days, peak ponding 40mm.

**The mic-drop diagnostic (ex-post cost evaluation):** I computed J ex-post on the actual trajectories using the documented weights:
```
              biomass    water  drought  ponding   TOTAL J
Hp=3  wet:    -0.978   +0.624   +0.043   +0.000  = -0.311  (BEST)
Hp=8  wet:    -0.953   +0.733   +0.003   +0.022  = -0.196
Hp=14 wet:    -0.898   +0.892   +0.009   +0.271  = +0.275  (WORST)
```
Hp=14 produces a worse cost-function value than Hp=3. A correctly-converged MPC with longer horizon and perfect forecast cannot do worse than a shorter-horizon MPC on the same objective — it could always replicate the shorter policy. This proves IPOPT is converging to local minima at Hp=14, not "smartly exploiting cheap ponding" (a wrong hypothesis Gemini and I both initially proposed but rejected with this evidence).
